# Docs

Esta pasta organiza a documentacao para dois perfis de leitura:

1. Tecnico: como o laboratorio foi desenhado, executado e validado.
2. Nao tecnico: qual problema de negocio e enderecado por resiliencia e alta disponibilidade.

## Conteudos Recomendados

- `TROUBLESHOOTING.md`: diagnostico de problemas comuns do laboratorio.
- `EVIDENCE_GUIDE.md`: guia de coleta de evidencias reais.

## Conteudos Opcionais para Evolucao

- `architecture.md`: topologia do cluster local e decisoes de design.
- `execution-guide.md`: passo a passo para executar cada cenario.
- `results-and-lessons.md`: resultados observados e licoes aprendidas.
- `recruiter-summary.md`: resumo executivo curto para avaliacao de perfil.

## Regras de Documentacao

1. Evitar jargao sem explicacao.
2. Informar contexto, acao, resultado e implicacao.
3. Referenciar evidencias em `evidence/` ao descrever resultados.
